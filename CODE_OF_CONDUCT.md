# Code of Conduct

`cartolina-js` contributors, maintainers, and users should be able to
participate without harassment or personal abuse.

## Expected Behavior

- Be direct about technical problems without making them personal.
- Assume review comments are about the work unless they say otherwise.
- Give enough context for others to reproduce bugs or understand decisions.
- Credit prior work when it informs a contribution.
- Respect maintainers' scope decisions, even when a feature is useful for a
  different project.

## Unacceptable Behavior

Unacceptable behavior includes:

- harassment, threats, or sustained disruption
- sexualized comments or unwelcome sexual attention
- insults, slurs, or demeaning comments about a person or group
- publishing private information without explicit permission
- bad-faith issue, review, or discussion activity
- using project spaces for unrelated political or commercial campaigns

## Scope

This code applies in project spaces, including issues, pull requests, wiki
edits, commits, discussions, demos, and project-related communication. It also
applies when someone is representing the project in public.

## Enforcement

Maintainers may remove comments, close issues, reject contributions, or limit
participation when conduct violates this document. Enforcement should be
proportionate to the behavior and the risk to other participants.

Report conduct problems to a maintainer. If a public issue would expose
private information or create more harm, use a private maintainer contact
channel when one is available. Reports should include links, screenshots, or
other concrete context when possible.

Maintainers handling a report should protect the reporter's privacy, avoid
unnecessary disclosure, and respond with the action they decide is appropriate
for the situation.

## Attribution

This document is based on the practical structure of the Contributor Covenant,
but is written for the current cartolina-js project rather than inherited from
the upstream `vts-browser-js` repository.
